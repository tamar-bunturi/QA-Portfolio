# TC-012 - Verify AI Search handles empty search queries

## Test Case Information

| Field         | Details                                         |
| ------------- | ----------------------------------------------- |
| Test Case ID  | TC-012                                          |
| Module        | AI Search                                       |
| Priority      | Medium                                          |
| Test Type     | Functional                                      |
| Preconditions | AI Search is enabled and accessible to the user |

---

## Test Steps

| Step | Action                                 | Expected Result                                         |
| ---- | -------------------------------------- | ------------------------------------------------------- |
| 1    | Navigate to the AI Search page         | AI Search interface is displayed                        |
| 2    | Click inside the search field          | Cursor is placed in the search field                    |
| 3    | Leave the search field empty           | No search term is entered                               |
| 4    | Click the Search button or press Enter | Search request is processed                             |
| 5    | Observe the system response            | Appropriate validation or default behavior is displayed |

---

## Expected Result

The system should handle empty search queries gracefully by either:

* Displaying a validation message requesting a search term, or
* Displaying a default state such as recent, recommended, or popular results.

The application should not return errors, crash, or display irrelevant results.
