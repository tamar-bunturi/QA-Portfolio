# TC-010 - Verify user can cancel an existing booking

## Test Case Information

| Field         | Details                                                       |
| ------------- | ------------------------------------------------------------- |
| Test Case ID  | TC-010                                                        |
| Module        | Booking System                                                |
| Priority      | High                                                          |
| Test Type     | Functional                                                    |
| Preconditions | User is logged into the application and has an active booking |

---

## Test Steps

| Step | Action                                     | Expected Result                                                        |
| ---- | ------------------------------------------ | ---------------------------------------------------------------------- |
| 1    | Navigate to My Bookings or Booking History | Booking list is displayed                                              |
| 2    | Locate an active booking                   | Booking is visible                                                     |
| 3    | Open the booking details                   | Booking information is displayed                                       |
| 4    | Click Cancel Booking                       | Cancellation confirmation is displayed                                 |
| 5    | Confirm the cancellation                   | Cancellation request is processed                                      |
| 6    | Refresh the booking list                   | Updated booking information is displayed                               |
| 7    | Verify booking status                      | Booking is marked as Cancelled or removed according to system behavior |

---

## Expected Result

The user should be able to cancel an existing booking successfully. The booking status should be updated accordingly and reflected throughout the application.
