# BUG-007 - Raw database error exposed to user when editing and saving offer price

| Field | Details |
|---|---|
| **Module** | Offer Management / Price Editing |
| **Category** | Functional |
| **Severity** | Critical |
| **Priority** | High |
| **Reproducibility** | 100% |

---

## Preconditions
1. User is authorized and logged in to the system
2. User is in Gardener Mode
3. User has already sent an offer to the customer

---

## Steps to Reproduce
1. Click on the "Sent" tab to filter sent offers
2. Find a pending offer and click on it to expand the price editing field
3. Modify the price by entering a new value (e.g. change 80 to 85)
4. Click the "Save" button
5. Observe the result

---

## Expected Result
When the gardener saves an edited price, the offer should be
updated successfully and the new price should be reflected
immediately. If the update fails for any reason, the system
should display a simple and clear user-friendly message such
as "Something went wrong. Please try again." with no raw
database or backend error details exposed to the user
under any circumstances

---

## Actual Result
When the gardener attempts to save an edited price on a
pending offer, a raw database error popup appears on the
screen

The price is not saved and the error message exposes internal
database function details directly to the end user, which is
both confusing and a security concern

---

## Impact
> This is a critical bug with both functional and security implications.
> From a functional perspective, gardeners are unable to edit their
> offer prices, which directly blocks a core business workflow on
> the platform. From a security perspective, exposing raw database
> function names, parameter types, and internal API structure to
> end users reveals sensitive backend architecture that could be
> exploited by malicious actors. This type of error exposure is
> considered a serious security vulnerability and should be treated
> as an urgent fix regardless of the underlying database issue.
