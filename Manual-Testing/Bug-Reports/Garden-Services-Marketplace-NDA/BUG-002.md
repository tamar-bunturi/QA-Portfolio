# BUG-002 - Guest mode request not visible in new user's MY REQUESTS section after registration

| Field | Details |
|---|---|
| **Module** | User Profile / My Requests |
| **Category** | UI |
| **Severity** | High |
| **Priority** | High |
| **Reproducibility** | 100% |

---

## Steps to Reproduce
1. Register in the system and log in
2. On the right panel, click on the profile icon
3. Open profile and click on "MY REQUESTS"

---

## Expected Result
After registration, the submitted request should be visible in the "MY REQUESTS" section of the user's profile

---

## Actual Result
After creating a new account, the user sees a confirmation message:
"Your request has been sent to the gardeners. You'll receive offers soon."
However, the request does not appear in the "MY REQUESTS" section of the user's profile

---

## Impact
> This is a high severity issue that directly impacts core business functionality.
> A customer who submitted a service request has no way to track, manage or 
> follow up on it after registration. The misleading confirmation message makes the 
> experience even more confusing, increasing the risk of user drop-off.
