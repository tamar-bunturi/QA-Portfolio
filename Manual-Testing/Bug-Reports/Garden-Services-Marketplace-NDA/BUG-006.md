# BUG-006 - Profile picture does not update after uploading a new photo

| Field | Details |
|---|---|
| **Module** | User Profile / Profile Picture |
| **Category** | Functional |
| **Severity** | Low |
| **Priority** | Low |
| **Reproducibility** | 100% |

---

## Preconditions
- User is authorized and logged in to the system
- User is on the Profile page

---

## Steps to Reproduce
1. Click on the camera icon on the profile picture
2. Select a new image from the device
3. Confirm or save the selection
4. Observe the profile picture

---

## Expected Result
After uploading a new image, the profile picture should update
immediately and display the newly selected photo. A success
message such as "Profile photo updated successfully" should
appear to confirm the action. If the upload fails for any
reason, a clear error message should be shown so the user
knows to try again

---

## Actual Result
After selecting and uploading a new image, the profile picture
does not update. The old photo or default avatar remains
displayed on the profile page with no visible change. No
success or error message is shown to inform the user whether
the upload was successful or failed

---

## Impact
> This bug affects the personal identity and trust of gardeners
> on the platform. A profile picture is one of the first things
> customers see when choosing a service provider.
