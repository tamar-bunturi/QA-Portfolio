# BUG-004 - Payment/order details banner persistently displayed at the top of the chat screen, covering the "Payment" button

| Field | Details |
|---|---|
| **Module** | UI / Chat Panel |
| **Category** | UI |
| **Severity** | High |
| **Priority** | Medium |
| **Reproducibility** | 100% |

---

## Preconditions
1. User is authorized and logged in to the system via Chrome Browser
2. User is in Gardener mode
3. User is on the Chat Panel

---

## Steps to Reproduce
1. Open any chat from the list
2. Observe the payment panel at the top of the screen

---

## Expected Result
The payment/order details banner should either be collapsible, dismissible, or only appear when relevant. It should not permanently occupy space at the top of the chat screen while the user is actively chatting

---

## Actual Result
A payment/order details banner showing amount, status and action button is persistently displayed at the top of the chat screen, taking up valuable screen space and cluttering the chat interface

---

## Impact
> This bug negatively affects the usability of the chat interface for gardeners.
> A permanently visible payment banner reduces the visible chat area, forcing 
> users to scroll more and making communication less efficient.
