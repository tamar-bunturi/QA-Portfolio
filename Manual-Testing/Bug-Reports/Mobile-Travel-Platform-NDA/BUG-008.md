# BUG-008 - Failed video upload blocks further message sending in chat

| Field | Details |
|---|---|
| **Module** | Messenger / Chat / Video Upload |
| **Category** | Functional |
| **Severity** | Major |
| **Priority** | High |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User has opened any personal chat

---

## Steps to Reproduce
1. Open a personal chat
2. Attempt to send a video
3. Wait until the upload transitions to a Failed status
4. Attempt to send a text message
5. Observe the result

---

## Expected Result
Even after a failed video upload, the user should be able to:
- Cancel the failed message or retry sending it (Retry)
- Freely send other text or media messages
- The chat should not enter a blocked state under any
  circumstances

---

## Actual Result
After a video upload failure, the chat becomes blocked. The
Send button remains in an active loading state (green spinner),
and sending any new text messages becomes impossible

---

## Impact
> This bug causes a complete and unrecoverable block of the
> chat functionality following a common network-related failure.
> A failed upload is a normal and expected event — poor
> connectivity, large file size, or server timeouts can all
> cause it. The application should handle this gracefully
> without affecting the user's ability to continue
> communicating. Instead, the chat enters a broken state
> with no way for the user to recover without restarting
> the conversation or the app entirely. This represents
> a critical failure in error handling and resilience,
> and should be treated as a high priority fix to ensure
> the messenger remains reliable under real-world
> network conditions.
