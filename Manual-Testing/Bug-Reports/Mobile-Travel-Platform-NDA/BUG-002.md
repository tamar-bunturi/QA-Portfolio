# BUG-002 - Messenger (Chat) icon color is inconsistent across screens and does not match the design

| Field | Details |
|---|---|
| **Module** | UI / Messenger Icon / Design Consistency |
| **Category** | UI |
| **Severity** | Low |
| **Priority** | Low |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
1. User is authorized and logged in to the system
2. User is on the Home page

---

## Steps to Reproduce
1. Observe the color of the Chat icon in the top right corner of the Home screen
2. Navigate to the Profile screen using the bottom navigation menu
3. Observe the color of the Chat icon on the Profile screen
4. Compare the icon colors across both screens

---

## Expected Result
The Messenger (Chat) icon color should be identical across all
relevant screens and should match the color defined in the
Figma design specifications

---

## Actual Result
The Messenger (Chat) icon color is inconsistent across different
screens and does not match the color defined in the Figma
design specifications

---

## Impact
> Although this is a low severity visual bug, it reflects a lack
> of design consistency across the application. Inconsistent icon
> colors create a perception of an unpolished and unfinished product,
> which can negatively affect user trust and brand identity.
> For a travel social platform where visual design plays a key role
> in user engagement, maintaining strict consistency with the
> approved Figma design specifications is essential to delivering
> a professional and cohesive user experience.
