# BUG-004 - Text jumps and behaves unstably when typing long messages in the chat input field

| Field | Details |
|---|---|
| **Module** | Messenger / Chat Input / Text Rendering |
| **Category** | Functional |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User has opened any personal or group chat

---

## Steps to Reproduce
1. Open any personal or group chat
2. Tap the message input field to open it
3. Start typing a long text message — especially one that spans multiple lines
4. Observe the text layout and cursor behavior during typing

---

## Expected Result
Text should be typed stably and smoothly. Line breaks should
occur naturally, the cursor should not jump unexpectedly, and
the message input field should not change the text layout
during the typing process

---

## Actual Result
While typing, the text layout becomes unstable — lines shift
unexpectedly, the cursor moves erratically, and the text
"jumps" during input, making the typing experience
uncomfortable and unreliable

---

## Impact
> This bug directly affects one of the most frequently used
> features of the application — sending messages. Unstable
> text rendering and cursor jumping during typing creates
> a frustrating and unprofessional user experience. Users
> who frequently send longer messages will encounter this
> issue repeatedly, which can lead to typing errors,
> loss of confidence in the messenger, and reduced overall
> engagement with the platform. For a social app where
> communication is central, a reliable and smooth typing
> experience is a fundamental quality requirement.
