# BUG-010 - "Deactivate" button is incorrectly positioned outside the Safe Area on the Account Settings screen

| Field | Details |
|---|---|
| **Module** | Account Settings / Login & Security / Deactivate |
| **Category** | UI / Layout |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User is on the Home screen

---

## Steps to Reproduce
1. Navigate to the Profile page from the bottom navigation menu
2. Open Settings / Privacy
3. Open Login & Security
4. Observe the "Deactivate" button

---

## Expected Result
The "Deactivate" button should have sufficient padding and margin
to be fully displayed within the Safe Area, complying with
platform UI Guidelines and matching the approved design
specifications

---

## Actual Result
The "Deactivate" button is positioned too close to the bottom
edge of the screen, falling outside the Safe Area / Gesture
Navigation Area. The button lacks the required bottom padding,
causing the UI to appear visually unbalanced

---

## Impact
> This is a UI layout bug that directly affects the accessibility
> and usability of a critical account management feature.
> The "Deactivate" button controls an irreversible account action -
> when it is positioned outside the Safe Area, it may be partially
> hidden or obscured by the device's gesture navigation bar,
> making it difficult or impossible to tap accurately on certain
> devices. This not only violates platform UI Guidelines but also
> creates a poor and unprofessional user experience on one of
> the most sensitive screens in the application.
