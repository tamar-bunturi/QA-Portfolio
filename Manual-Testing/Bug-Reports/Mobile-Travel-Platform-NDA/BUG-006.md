# BUG-006 - Reply and Like functions do not work on photo and video messages

| Field | Details |
|---|---|
| **Module** | Messenger / Chat / Message Actions |
| **Category** | Functional |
| **Severity** | Major |
| **Priority** | High |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User has opened any personal or group chat

---

## Steps to Reproduce
1. Open any personal or group chat
2. Send or receive a photo or video message
3. Attempt to use the Reply function on the photo or video message
4. Attempt to use the Like (emoji reaction) function on the
   photo or video message
5. Observe the result

---

## Expected Result
The Reply and Like functions should work on all supported
message types, including photo and video messages. According
to the requirements, Reply is described as a core chat action
and media messages are defined as supported message types

---

## Actual Result
The Reply and Like functions are not available or cannot be
completed on photo and video messages. These actions work
on text messages but fail on media message types

---

## Impact
> This bug blocks core interaction features on media messages,
> which are among the most frequently shared content types in
> a travel social application. The inability to reply to or
> react to photo and video messages creates an inconsistent
> and incomplete messaging experience — users can interact
> with text messages but not with media, which feels broken
> and unpredictable. For a platform where visual content
> sharing is central to the user experience, ensuring that
> all message interaction features work consistently across
> all message types is a fundamental quality requirement.
