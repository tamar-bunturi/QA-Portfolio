# BUG-009 - Group member avatars are incorrectly laid out in Chat Settings for 3+ members

| Field | Details |
|---|---|
| **Module** | Messenger / Group Chat / Chat Settings / Avatar Layout |
| **Category** | UI / Layout |
| **Severity** | Minor |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User has created or is a member of a group chat with 3 or more members

---

## Steps to Reproduce
1. Create a group chat with 3 or more members
2. Open Chat Settings
3. Observe the group avatar layout at the top of the screen
4. Compare the layout with a 2-member group chat for reference

---

## Expected Result
Regardless of the number of group members, avatars should be
displayed correctly and consistently. No extra placeholder
circles should appear and no elements should overflow outside
the screen boundaries. The UI should be visually uniform and
properly structured for all group sizes

---

## Actual Result
When a group has 3 or more members, the group avatars are
incorrectly positioned. An extra grey placeholder circle
appears and the avatars overlap each other incorrectly,
causing the UI to look inconsistent and visually broken

---

## Impact
> Although this is a minor severity UI bug, it represents a
> layout regression that becomes visible in one of the most
> common group chat scenarios — having 3 or more participants.
> Incorrectly rendered avatars and unexpected placeholder
> elements create a perception of an unpolished and untested
> interface. For a social platform where group chats are a
> frequently used feature, maintaining visual consistency
> across all group sizes is important for delivering a
> professional and trustworthy user experience.
