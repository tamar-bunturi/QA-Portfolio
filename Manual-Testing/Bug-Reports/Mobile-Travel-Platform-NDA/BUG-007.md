# BUG-007 - Navigating to user profile from chat is not possible

| Field | Details |
|---|---|
| **Module** | Messenger / Chat / User Profile Navigation |
| **Category** | Functional |
| **Severity** | Major |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User has opened any personal chat

---

## Steps to Reproduce
1. Open a personal chat on an Android device
2. Tap on the other user's name or profile photo within the chat
3. Observe the result
4. Repeat the same steps on an iOS device for comparison

---

## Expected Result
Tapping on a user's name or profile photo inside a chat should
navigate to that user's profile page on Android, just as it
does on iOS. Both platforms should have consistent and identical
functionality for this feature

---

## Actual Result
On Android, tapping on the user's name or profile photo inside
the chat does not navigate to the user's profile. The same
action on iOS successfully opens the user's profile page

---

## Impact
> This is a cross-platform consistency bug that creates an
> unequal experience between Android and iOS users. Feature
> parity between platforms is a fundamental quality standard
> for any mobile application. When a core navigation feature
> works on one platform but not the other, it signals a gap
> in platform-specific implementation and testing coverage.
> For users on Android, the inability to view a chat
> participant's profile directly from the conversation
> reduces context and disrupts natural social interaction
> flows. This issue is particularly important to resolve
> before any public release to ensure a consistent and
> professional experience across all supported platforms.
