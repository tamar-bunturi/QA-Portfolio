# BUG-011 - "Login & Security" element text does not match the capitalization of other menu elements on the Settings screen

| Field | Details |
|---|---|
| **Module** | Settings / Privacy / Login & Security |
| **Category** | UI / Text Consistency |
| **Severity** | Low |
| **Priority** | Low |
| **Reproducibility** | 100% |

---

## Preconditions
- User is authorized and logged in to the system
- User is on the Home screen

---

## Steps to Reproduce
1. Navigate to the Profile page from the bottom navigation menu
2. Open Settings / Privacy
3. Observe the "login & security" menu element text

---

## Expected Result
All menu elements should use a consistent text format.
"Login & Security" should be displayed in Title Case format
to maintain visual consistency with other menu elements
and comply with UI design standards

---

## Actual Result
The "Login & Security" element is displayed in incorrect text
format as "login & security" in lowercase, while all other
menu elements use Title Case format

---

## Impact
> Although this is a low severity visual bug, text case
> inconsistency across UI elements reflects a lack of attention
> to detail in the implementation of design standards. When one
> menu item uses lowercase while all others follow Title Case,
> it creates a visually inconsistent and unpolished interface.
> For a mobile application where UI consistency directly
> contributes to the perception of product quality and
> professionalism, even minor text formatting issues should
> be addressed to ensure a cohesive and trustworthy
> user experience.
