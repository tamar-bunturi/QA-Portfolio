# BUG-001 - Messenger (Chat) icon is missing on the Explore screen

| Field | Details |
|---|---|
| **Module** | UI / Explore Screen / Navigation |
| **Category** | UI |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User is on the Home page

---

## Steps to Reproduce
1. Observe the top right corner of the Home screen - the Chat icon is visible
2. Navigate to the Explore page using the bottom navigation menu
3. Observe the top right corner of the Explore screen

---

## Expected Result
The Messenger (Chat) icon should be visible on the Explore screen,
allowing the user to navigate to the Messenger directly from the
Explore screen, as described in the requirements

---

## Actual Result
The Messenger (Chat) icon is not displayed on the Explore screen.
As a result, the user is unable to access the Messenger from
this screen

---

## Impact
> This bug breaks the navigation consistency of the application.
> According to the requirements, the Messenger icon should be
> accessible from the Explore screen just as it is from the Home
> and Profile screens. When a key navigation element is missing
> from one screen but present on others, it creates a confusing
> and inconsistent user experience. Users who are actively
> exploring content may miss messages or be forced to navigate
> away from their current screen just to access the Messenger,
> which disrupts their natural app flow.
