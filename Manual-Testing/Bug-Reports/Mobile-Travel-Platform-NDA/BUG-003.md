# BUG-003 - Typed text disappears from chat input field after sending a photo

| Field | Details |
|---|---|
| **Module** | Messenger / Chat Input / Photo Attachment |
| **Category** | Functional |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User has opened any personal or group chat

---

## Steps to Reproduce
1. Open any personal or group chat
2. Type any text in the message input field
3. Without sending the text, tap the photo attachment (Attach) button
4. Select a photo from the gallery
5. Send the photo
6. Observe the message input field

---

## Expected Result
After sending the photo, the previously typed text should not
be lost. It should either:
- Remain in the message input field, or
- Be sent together with the photo if the application design
supports combined messages

---

## Actual Result
The photo is sent successfully, however the text that was
previously typed in the input field disappears and the
input field becomes empty

---

## Impact
> This bug causes unexpected data loss during a common messaging
> workflow. When a user types a message and decides to attach a
> photo before sending, they reasonably expect their typed text
> to be preserved. Losing typed content without warning is a
> frustrating and confusing experience that breaks the user's
> trust in the reliability of the messenger. For a social platform
> where communication is a core feature, any data loss in the
> messaging flow - even of unsent text - should be treated as
> a functional issue requiring a fix.
