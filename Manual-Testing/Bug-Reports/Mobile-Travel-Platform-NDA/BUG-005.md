# BUG-005 - File sending is completely broken in chat - no file types can be sent

| Field | Details |
|---|---|
| **Module** | Messenger / Chat / File Attachment |
| **Category** | Functional |
| **Severity** | Major |
| **Priority** | High |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Android |

---

## Preconditions
- User is authorized and logged in to the system
- User has opened any personal or group chat

---

## Steps to Reproduce
1. Open the application and log in
2. Open any personal or group chat
3. Tap the file attachment (Attach) button
4. Select any supported file type (e.g. .pdf, .doc, .docx, .txt)
5. Attempt to send the file
6. Observe the result

---

## Expected Result
The user should be able to successfully send files in the chat.
According to the application requirements, file sending is a
supported feature of the messenger

---

## Actual Result
No file type can be sent in the chat. Sending fails for all
tested file formats including .pdf, .doc, .docx, and .txt

---

## Impact
> This is a high severity bug that completely blocks a documented
> core feature of the messenger. File sharing is a fundamental
> communication capability that users rely on for sharing travel
> documents, itineraries, booking confirmations, and other
> travel-related content. The fact that no file type works
> suggests a systemic failure rather than an isolated edge case,
> making this a critical gap between the documented requirements
> and the actual functionality delivered. This bug should be
> treated as a priority fix before the feature is considered
> ready for production release.
