# BUG-001 - System Treats Adding a New Photo as Changing the Profile Photo

| Field | Details |
|---------|---------|
| Module | Profile Management / Photos |
| Category | Functional |
| Severity | Medium |
| Priority | High |
| Reproducibility | 100% |

---

## Preconditions

- User is logged into the application.
- User already has an existing profile photo.
- User profile contains at least 1 profile photo.
- User has access to the device gallery.

---

## Steps to Reproduce

1. Open the application.
2. Navigate to **Profile**.
3. Tap **Edit Profile**.
4. Tap the **"+" Add Photo** button.
5. Select a new image from the gallery.
6. Confirm the upload.
7. Return to the profile page.

---

## Expected Result

The selected image should be added to the user's photo gallery while keeping the current profile photo unchanged.

The profile photo should only change when the user explicitly chooses **Set as Profile Photo**.

---

## Actual Result

The system automatically replaces the current profile photo with the newly uploaded image instead of adding it as an additional gallery photo.

---

## Impact

Users may unintentionally lose their preferred profile photo while attempting to add more photos to their profile.

This behavior creates confusion and negatively impacts profile customization.
