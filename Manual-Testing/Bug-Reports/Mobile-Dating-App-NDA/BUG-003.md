# BUG-003 - Support Chat Header Appears Clickable but Has No Functionality

| Field           | Details        |
| --------------- | -------------- |
| Module          | Support / Chat |
| Category        | UI / Usability |
| Severity        | Low            |
| Priority        | Medium         |
| Reproducibility | 100%           |

---

## Preconditions

* User is logged into the application.
* User has opened the Settings page.

---

## Steps to Reproduce

1. Open the App.
2. Navigate to **Settings**.
3. Open the **Support** section.
4. Send a message and wait for a response.
5. Observe the **"Support"** element displayed in the top center of the chat screen.
6. Tap on the element.

---

## Expected Result

The element should either:

* Be visually styled as a non-interactive title if no action is available, or
* Open the corresponding functionality when tapped.

Users should be able to clearly distinguish between clickable and non-clickable elements.

---

## Actual Result

The **"Support"** element is visually presented as an interactive button or profile header, creating the expectation that it can be tapped.

However, tapping the element produces no response and no action is performed.

---

## Impact

The inconsistent behavior may confuse users and lead them to believe that a feature is unavailable, broken, or not responding.

The interface creates a false expectation of interactivity, negatively affecting usability and user experience.

---

## Notes

In regular user chats, a similar header element is functional and provides access to actions such as viewing a profile, deleting a conversation, or blocking a user.

Since these actions are not available within the Support chat, the visual design of the header should be adjusted to clearly indicate that it is a static title rather than an interactive element.
