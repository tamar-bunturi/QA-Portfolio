# BUG-003 - Adding a New Profile Photo Triggers Unnecessary Re-Verification as Profile Picture Change

| Field           | Details                    |
| --------------- | -------------------------- |
| Module          | Profile / Photo Management |
| Category        | Functional                 |
| Severity        | Medium                     |
| Priority        | Medium                     |
| Reproducibility | 100%                       |

---

## Preconditions

* User is logged into the application.
* User is on the Profile page.

---

## Steps to Reproduce

1. Open the App.
2. Navigate to the Profile page.
3. Tap on the current profile photo.
4. Tap the "+" button.
5. Upload a second/additional photo.

---

## Expected Result

The new photo should be added successfully as an additional profile photo without affecting the existing profile picture.

If verification is required for adding new photos, the user should receive a clear and accurate message explaining the reason for the verification request.

---

## Actual Result

When uploading a second photo, the system automatically treats the action as a profile picture change and triggers the verification process again.

This behavior may confuse users because they are adding an additional photo rather than replacing their primary profile picture.

---

## Impact

Users may incorrectly assume that their profile picture has been changed or that an error has occurred.

The unexpected verification request creates confusion and may negatively affect the user experience.

**Note:** If re-verification is intended when adding additional photos, the application should display a clear and informative message, such as:

*"You have added a new photo. For security reasons, please complete verification."*
