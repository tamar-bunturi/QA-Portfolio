# BUG-001 - Formatting toolbar overlaps AI action dropdown panel during text modification in the editor

| Field | Details |
|---|---|
| **Module** | Editor / Formatting Toolbar |
| **Category** | UI |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |

---

## Preconditions
1. User is authorized and logged in to the system
2. A document is open in the editor
3. The document contains text

---

## Steps to Reproduce
1. Select a portion of text in the editor
2. Click the attachment icon from the formatting toolbar
3. Observe the opened dropdown panel

---

## Expected Result
The dropdown panel should appear below the document content layer
with correct z-index, so the user can clearly see and interact
with all available options without any overlap or obstruction

---

## Actual Result
When clicking the attachment icon, the dropdown panel appears
and overlaps the formatting toolbar and surrounding document content.
The user cannot clearly see what function is currently active on
the toolbar, as the dropdown covers the formatting panel entirely

---

## Impact
> This bug disrupts the core editing workflow of the platform.
> When the formatting toolbar and AI dropdown panel overlap,
> users lose visibility of their active formatting options and
> cannot efficiently interact with either panel simultaneously.
