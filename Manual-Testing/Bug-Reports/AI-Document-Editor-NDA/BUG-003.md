# BUG-003 - System fails to delete more than 10 selected projects simultaneously

| Field | Details |
|---|---|
| **Module** | Project Management / Bulk Delete |
| **Category** | Functional |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |

---

## Preconditions
User is authorized and logged in to the system with 13 or more projects added

---

## Steps to Reproduce
1. Open the web application
2. Select 13 projects simultaneously, for example using Ctrl + A
3. Try to delete the selected files one of the following ways:
   - Right-click on any selected project and choose Delete
   - Click the three-dot menu next to any project and choose Delete
4. Confirm deletion in the Confirmation Pop-up

---

## Expected Result
After confirming deletion, all 10+ selected files should be
deleted successfully and should no longer appear in the file list.
The system should display a success confirmation message

---

## Actual Result
Before the deletion is confirmed, more than 10 of the selected
files are not deleted. The system displays the following
error message: "Failed to delete items". Despite the error,
the selected files still remain in the file list

---

## Impact
> This bug limits a core file management feature of the platform,
> forcing users to delete projects one by one when working with
> large volumes of documents. For an AI-powered document editor
> used by professional users who may manage dozens of manuscripts
> and projects simultaneously, the inability to perform bulk deletion
> significantly reduces productivity. The generic "Failed to delete items"
> error message also provides no guidance to the user, making it
> impossible to understand or work around the limitation.
