# BUG-005 - AI Assistant model selection is not persisted after saving and reverts to Default AI Assistant

| Field | Details |
|---------|---------|
| Module | AI Assistant Settings |
| Category | Functional |
| Severity | Medium |
| Priority | Medium |
| Reproducibility | 100% |

---

## Preconditions

- User is logged into the application.
- AI Assistant Settings page is accessible.
- Default AI Assistant is configured as Sonnet 4.6.

---

## Steps to Reproduce

1. Navigate to the application.
2. Open **Settings**.
3. Open **AI Assistant Settings**.
4. Change the configured model from the default model (Sonnet 4.6) to another available model (e.g., Opus 4.6).
5. Click **Save**.
6. Refresh the page or reopen AI Assistant Settings.
7. Start a new AI Assistant session.
8. Observe the active AI model.

---

## Expected Result

The selected AI Assistant model should be successfully saved and remain active after refreshing the page, reopening settings, or creating a new AI Assistant session.

---

## Impact

- Users cannot reliably change the AI model configuration.
- Saved preferences are ignored.
- AI responses may be generated using an unintended model.
- Creates confusion regarding which model is currently active.
