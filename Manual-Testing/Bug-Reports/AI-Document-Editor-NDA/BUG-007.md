# BUG-007 - Deleted files are displayed in Library search results

| Field | Details |
|---------|---------|
| Module | Library Search |
| Category | Functional |
| Severity | Medium |
| Priority | Medium |
| Reproducibility | 100% |

---

## Preconditions

1. User is logged into the application.
2. A document named ISTQB exists in the Library.
3. The document ISTQB has been deleted and moved to the Bin/Trash.
4. Another active document related to ISTQB remains available in the Library.
  
---

## Steps to Reproduce

1. Navigate to the application.
2. Open the Library page.
3. Verify that the document is moved to the Bin/Trash.
4. Return to the Library search page.
5. Enter "ISTQB" in the search f  ield.
6. Review the search results.

---

## Expected Result

Only active documents available in the Library should be returned in the search results.
Documents that have been deleted and moved to the Bin/Trash should not appear in Library search results.

---

## Actual Result

The search results display both:

The active ISTQB document.
The deleted ISTQB document located in the Bin/Trash.

The system includes deleted files in Library search results even though they are no longer active.

---

## Impact

Users may open or reference deleted documents by mistake.
Search results become confusing and unreliable.
Deleted content appears alongside active content.
Users cannot easily distinguish between active and deleted documents.
