# BUG-004 - PDF file cannot be moved to a folder using any method despite system displaying a success message

| Field | Details |
|---|---|
| **Module** | File Management / Folder Organization |
| **Category** | Functional |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |

---

## Preconditions
User is authorized and logged in to the system

---

## Steps to Reproduce
1. From the left menu click "+" and create a folder
2. From the left menu click "+" and upload any PDF file
3. From the left menu click "+" and create a text file
4. Try to move the text file to the folder using Drag and Drop
5. Try to move the PDF file to the folder using Drag and Drop
6. Try to move the PDF file to the folder using Copy & Paste
7. Try to move the PDF file to the folder using Cut & Paste

---

## Expected Result
The PDF file should be moveable to a folder using Drag and Drop
or any other available method. If the move is successful, the file
should appear inside the folder and be removed from its original
location. If the move is not supported, the system should display
a clear error message - not a success confirmation

---

## Actual Result
The PDF file cannot be moved to the folder using any method -
neither Drag and Drop nor any other available option. However,
the system displays a success message indicating the file was
moved to the folder. When the folder is opened, the PDF file
is not there and it remains in its original location.
Note: The text file moves to the folder correctly
using Drag and Drop

---

## Impact
> This bug combines two serious issues: a broken core feature and
> a misleading success message. When users are told an action was
> completed successfully but the actual result is the opposite,
> it destroys trust in the platform's reliability. Users may spend
> significant time searching for files they believe were organized,
> not realizing the move never happened.
