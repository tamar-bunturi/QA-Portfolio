# BUG-005 - Email Link Does Not Trigger Mail Client When Clicked

| Field | Details |
|---------|---------|
| Module | Contact Section / Footer |
| Category | Functional |
| Severity | Medium |
| Priority | Medium |
| Reproducibility | 100% |

---

## Preconditions

- User has access to the website.
- User is on a page containing the contact information section.

---

## Steps to Reproduce

1. Open the website.
2. Navigate to the Contact section or Footer.
3. Locate the available contact methods.
4. Click the **Email** link.
5. Observe the behavior.

---

## Expected Result

Clicking the **Email** link should trigger the default mail application or open a new email draft using the appropriate `mailto:` functionality.

Users should be able to start composing an email directly from the website.

---

## Actual Result

The **Email** element appears to be interactive; however, clicking it does not trigger any action.

No mail client is opened and no email draft is created.

---

## Impact

Users are unable to contact the organization through the provided email communication channel.

This may lead to missed inquiries, reduced accessibility, and a poor user experience.
