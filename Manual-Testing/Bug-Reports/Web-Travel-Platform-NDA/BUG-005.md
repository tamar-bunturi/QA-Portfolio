# BUG-004 - Progress indicator displays incorrect step number on Review & Confirm page

| Field | Details |
|---|---|
| **Module** | Property Creation / Progress Indicator |
| **Category** | UI |
| **Severity** | Low |
| **Priority** | Low |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Web (Google Chrome 150.0.7871.128 / Windows 11 Pro) |

---

## Preconditions
- User is logged in to the application.
- User has progressed through the property creation flow to the **Review & Confirm** page.

---

## Steps to Reproduce
1. Navigate to the **Review & Confirm** page.
2. Observe the progress indicator displayed in the top-right corner.

---

## Expected Result
The progress indicator should accurately reflect the user's current step in the property creation process.

For example:
- If **Review & Confirm** is Step **8/8**, it should display **Step 8/8**.
- If **Review & Confirm** is Step **9/9**, it should display **Step 9/9**.

The displayed step count should always match the actual number of steps in the workflow.

---

## Actual Result
The progress indicator displays **Step 9/8**, indicating an impossible step count where the current step exceeds the total number of steps.

---

## Impact
> An incorrect progress indicator creates confusion about the user's position within the property creation process and reduces confidence in the workflow. Although this issue does not block functionality, it negatively affects usability and gives the impression of an incomplete or poorly validated interface.
