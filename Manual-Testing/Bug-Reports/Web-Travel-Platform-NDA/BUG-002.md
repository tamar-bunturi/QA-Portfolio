# BUG-001 - Optional Star Rating field displays technical validation message

| Field | Details |
|---|---|
| **Module** | Property Creation |
| **Category** | Validation / UI |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Web (Google Chrome 150.0.7871.128 / Windows 11 Pro) |

---

## Preconditions
- User is logged in to the application.
- User is on the **Create Property** page.
- Required fields (Property Name, Property Type, and Description) are completed.
- **Star Rating (Optional)** is left empty.

---

## Steps to Reproduce
1. Navigate to the **Create Property** page.
2. Fill in all required fields.
3. Leave the **Star Rating (Optional)** field empty.
4. Click the **Next** button.

---

## Expected Result
A clear, user-friendly validation message should be displayed.

Example:
> "Star Rating must be a number."

The message should use the same terminology as the UI and should not expose internal implementation details.

---

## Actual Result
A technical validation message is displayed:

> "starRating must be a number"

The message exposes the internal field name (`starRating`) in camelCase instead of displaying a user-friendly label.

---

## Impact
> Users are presented with a developer-oriented validation message that exposes internal implementation details. This reduces the professionalism of the interface, may confuse non-technical users, and creates an inconsistent validation experience. Error messages should use human-readable field names that match the UI.
