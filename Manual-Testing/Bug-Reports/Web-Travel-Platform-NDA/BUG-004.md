# BUG-003 - Date of Birth is displayed in ISO 8601 format instead of a user-friendly date format

| Field | Details |
|---|---|
| **Module** | Ownership & Legal Information |
| **Category** | UI / Data Formatting |
| **Severity** | Low |
| **Priority** | Low |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Web (Google Chrome 150.0.7871.128 / Windows 11 Pro) |

---

## Preconditions
- User is logged in to the application.
- User has completed the property creation process.
- Ownership & Legal Information contains a valid Date of Birth.

---

## Steps to Reproduce
1. Navigate to the **Ownership & Legal Information** section.
2. Click the **Edit** button.
3. Observe the displayed **Date of Birth** value.

---

## Expected Result
The **Date of Birth** should be displayed in a user-friendly, localized format.

---

## Actual Result
The **Date of Birth** is displayed in the ISO 8601 format:

> `1988-11-06T00:00:00.000Z`

This appears to expose the raw value received from the backend/API instead of a properly formatted date for end users.

---

## Impact
> Displaying raw ISO 8601 timestamps negatively affects readability and creates an unpolished user experience. End users expect dates to be presented in a familiar, localized format rather than a backend-oriented representation. This inconsistency reduces the overall usability and professionalism of the interface.
