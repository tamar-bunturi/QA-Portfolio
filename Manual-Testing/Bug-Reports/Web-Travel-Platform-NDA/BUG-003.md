# BUG-002 - "Go Back" button does not navigate to the previous step on Property Location page

| Field | Details |
|---|---|
| **Module** | Property Creation / Property Location |
| **Category** | Navigation |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Web (Google Chrome 150.0.7871.128 / Windows 11 Pro) |

---

## Preconditions
- User is logged in to the application.
- User has reached the **Property Location** step during the property creation process.

---

## Steps to Reproduce
1. On the **Property Location** page, click **Place a Pin**.
2. Click the **Go Back** button.

---

## Expected Result
The **Go Back** button should navigate the user to the previous step of the property creation flow while preserving any previously entered information.

---

## Actual Result
Clicking the **Go Back** button does not return the user to the previous step. Instead:
- The user remains on the current page.
- Navigation does not occur.
- The button appears unresponsive and provides no visual or functional feedback.

---

## Impact
> This issue breaks the expected navigation flow during property creation. Users cannot return to the previous step to review or modify previously entered information, which interrupts the workflow and may force them to restart the process. A non-functional navigation button also reduces user confidence and creates an inconsistent experience.
