# BUG-001 - Technical validation error exposes internal field name when Star Rating is left empty

| Field | Details |
|---|---|
| **Module** | Property Creation / Validation |
| **Category** | Validation / Error Handling |
| **Severity** | Medium |
| **Priority** | Medium |
| **Reproducibility** | 100% |
| **Status** | New |
| **Platform** | Web (Google Chrome 150.0.7871.128 / Windows 11 Pro) |

---

## Preconditions
- User is logged in to the application
- User has navigated to the **Create Property** page
- Property Name, Property Type, and Description are filled in
- The **Star Rating (Optional)** field is left empty

---

## Steps to Reproduce
1. Open the **Create Property** page.
2. Fill in all required property information.
3. Leave the **Star Rating (Optional)** field empty.
4. Click the **Next** button.

---

## Expected Result
A user-friendly validation message should be displayed if validation fails.

Example:
> "Star Rating must be a number."

The message should be properly formatted and should not expose internal field names or implementation details.

---

## Actual Result
A technical validation message is displayed:

> "starRating must be a number"

The message exposes the internal field name (`starRating`) in camelCase, which is intended for developers rather than end users.

---

## Impact
> The application exposes internal validation details instead of displaying a user-friendly error message. Showing implementation-specific field names creates an inconsistent user experience, may confuse non-technical users, and reduces the overall quality and professionalism of the interface. Validation messages should be written in clear, human-readable language without revealing internal object or variable names.
