# BUG-004 - Customer chat sessions are not synchronized across multiple browser tabs

| Field | Details |
|---------|---------|
| Module | Shopify Integration |
| Category | Functional |
| Severity | High |
| Priority | High |
| Reproducibility | 100% |

## Preconditions

- Customer chat widget is opened in two separate browser tabs/windows using the same browser session.
- Admin Panel is open and connected to the conversation.
- Both chat windows belong to the same customer session.

---

## Steps to Reproduce

1. Open Customer Chat Widget in Browser Tab 1.
2. Open the same Customer Chat Widget in Browser Tab 2.
3. Send a message from Customer Chat Window 1.
4. Verify the message appears in the Admin Panel.
5. Send a reply from the Admin Panel.
6. Observe both customer chat windows.
7. Repeat the process using Customer Chat Window 2.
8. Compare the conversation history displayed in both tabs.

---

## Expected Result

All customer chat windows belonging to the same user session should remain synchronized. Messages sent from the Admin Panel should appear in every active customer chat window. Conversation history should remain identical across all open chat instances, and real-time updates should be delivered without requiring a page refresh.

---

## Actual Result

Both customer chat windows can independently send messages to the Admin Panel, and the Admin Panel successfully receives messages from both instances. However, replies sent from the Admin Panel are delivered to only one customer chat window. The second chat window does not receive updates in real time, resulting in inconsistent conversation histories between tabs representing the same user session.

---

## Impact

This issue creates inconsistent communication between customers and support agents. Users may miss important replies, see outdated conversation histories, or become confused when different browser tabs display different messages despite representing the same session. The reliability of the chat experience is significantly reduced.
